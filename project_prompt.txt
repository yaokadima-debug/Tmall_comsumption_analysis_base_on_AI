【项目概述】

请为我构建一个完整的天猫用户销售数据分析平台。项目中已有一个MySQL数据库 `tmall_data`，包含5张表，总计约57,000行数据。请从数据库提取数据，完成数据分析、机器学习聚类、BI可视化报表和分析报告生成。

项目包含统一入口脚本 main.py，可一键运行全部7个分析脚本。

【数据库连接信息】
- 数据库类型: MySQL 8.0
- 主机: localhost
- 端口: 3306
- 用户名: root
- 密码: 请查看 user_auth_db.py 文件并修改为实际密码
- 数据库名: tmall_data
- 字符集: utf8mb4
- Python连接库: pymysql
- IMPORTANT: 数据库密码已提取到 user_auth_db.py，请修改该文件中的 DB_CONFIG['password'] 为实际密码后运行所有脚本

【数据库表结构】

表1: users (用户基础信息表, 5,000行)
字段: user_id VARCHAR(255), age INT, gender VARCHAR(255), province VARCHAR(255), city VARCHAR(255), registration_date VARCHAR(255), member_level VARCHAR(255), account_balance DOUBLE, credit_score INT
说明: member_level取值: 普通/铜牌/银牌/金牌/钻石; credit_score范围: 400-800; age范围: 18-63

表2: orders (订单表, 15,000行)
字段: order_id VARCHAR(255), user_id VARCHAR(255), product_id VARCHAR(255), quantity INT, order_date VARCHAR(255), order_date_date DATE, order_status VARCHAR(255), payment_method VARCHAR(255), unit_price DOUBLE, total_amount DOUBLE, discount DOUBLE, actual_payment DOUBLE, delivery_date VARCHAR(255), receive_date VARCHAR(255), review_score INT, review_content VARCHAR(255)
说明: order_status取值: 待付款/已付款/已发货/已收货/已完成/已取消/已退款; payment_method取值: 支付宝/微信/银行卡/信用卡/花呗; review_score范围: 4-5; order_date和order_date_date为同一日期不同格式

表3: products (产品表, 2,000行)
字段: product_id VARCHAR(255), product_name VARCHAR(255), category VARCHAR(255), brand VARCHAR(255), price DOUBLE, sales_count INT
说明: category有15个品类; brand有113个品牌

表4: user_behaviors (用户行为表, 30,000行)
字段: behavior_id VARCHAR(255), user_id VARCHAR(255), product_id VARCHAR(255), behavior_type VARCHAR(255), behavior_time DATETIME, duration_seconds INT
说明: behavior_type取值: 浏览/点击/收藏/加购

表5: user_features (用户特征表, 5,000行)
字段: user_id VARCHAR(255), total_spent DOUBLE, order_count DOUBLE, avg_order_amount DOUBLE, browse_count DOUBLE, click_count DOUBLE, favorite_count DOUBLE, cart_count DOUBLE, days_since_last_order INT, order_frequency DOUBLE, repurchase_indicator INT, purchase_intent DOUBLE, consumption_level VARCHAR(255), member_level_score INT
说明: consumption_level取值: 高/中/低; repurchase_indicator: 0/1

功能: 用户聚类分析独立脚本

要求:
- 从MySQL提取users + user_features表数据
- RFM特征工程: 计算R值(距今天数=365-距上次下单天数标准化)、F值(订单数)、M值(累计消费)
- 特征矩阵构建: 选择12个数值特征(R/F/M + browse_count/click_count/favorite_count/cart_count/avg_order_amount/order_frequency/purchase_intent/member_level_score/age)

- K-Means聚类: K=2到8遍历，记录SSE和Silhouette Score，自动选择最佳K(目标K=5，通过肘部法则+轮廓系数验证)
- PCA降维到2维用于可视化
- 聚类命名: 使用RFM排序法(R-rank + F-rank + M-rank综合分)确保5个群体获得唯一且有业务含义的名称:
  * 高价值忠诚用户(高F高M低R)
  * 活跃发展中用户(中F中M中R)
  * 潜力唤醒用户(中低F中低M中高R)
  * 低频待激活用户(低F低M中R)
  * 流失预警用户(低F低M高R)
- 生成HTML聚类报告(含肘部法则图、轮廓系数图、PCA散点图、群体分布图、RFM雷达图、特征表、运营建议)
- 聚类结果写入MySQL: 创建表 claude_user_clusters(user_id, cluster, cluster_name, R值, F值, M值, PCA_X, PCA_Y)，插入5000条
- 使用 random_state=42 确保可复现
- 输出: 终端打印群体统计 + HTML报告 + 数据库写入

【关键技术要点】

1. 日期处理:
- orders表中有两个日期字段: order_date(VARCHAR格式)和order_date_date(DATE格式)，优先使用order_date_date
- registration_date为VARCHAR格式，需使用pd.to_datetime转换
- behavior_time为DATETIME格式

2. 数据聚合:
- GMV = SUM(orders.total_amount)
- 实际营收 = SUM(orders.actual_payment)
- 折扣率 = SUM(discount) / SUM(total_amount)
- 客单价ARPU = 实际营收 / 有订单的用户数
- 复购率 = 订单数≥2的用户数 / 总用户数

3. 用户生命周期计算:
- DAU: 每日有行为记录的去重用户数(从user_behaviors表behavior_time按日聚合)
- MAU: 每月有行为记录的去重用户数
- 粘性比: DAU均值 / MAU均值
- 月新增用户: 当月首次下单的用户数(从orders表按order_date_date聚合)
- 月流失用户: 当月无行为但前30天有行为的用户数
- 留存队列: 按用户首单周分组(Cohort)，计算每队列在第0/1/2/4/8/12周仍有行为的用户占比
- 复购间隔: 同一用户相邻订单的order_date_date天数差
- 生命周期阶段:
  * 新用户: 首次下单≤30天
  * 活跃用户: 最近30天有行为
  * 沉默用户: 30-90天无行为
  * 流失用户: >90天无行为

4. RFM计算:
- R(Recency): 距今天数，用固定参考日(如2025-12-31或数据最大日期)减去用户最后下单日期
- F(Frequency): 用户历史订单总数
- M(Monetary): 用户累计消费金额(actual_payment求和)

5. 聚类:
- random_state=42 必须固定
- K值选择逻辑: 计算K=2到8的Silhouette Score和SSE，选择Silhouette最高且业务合理的K(预期K=5)
- 聚类命名必须使用RFM综合排序法，确保5个群体名称不重复

6. HTML Tab切换CSS核心逻辑:
```css
.tab-content { display: none; }
#tab-0:checked ~ .tab-content-0,
#tab-1:checked ~ .tab-content-1,
... { display: block; }
```

7. Plotly图表在隐藏Tab中可能尺寸异常，解决方式:
- 所有图表在页面加载时一次性渲染
- Tab切换时调用 Plotly.Plots.resize() 或 new Promise(resolve => setTimeout(resolve, 100)).then(() => Plotly.Plots.resize())

8. python-docx表格样式:
- 使用 'Table Grid' 样式
- 表头背景色 #667eea，字体白色加粗
- 数据行交替 #F8F9FF / #FFFFFF
- 表格居中对齐
